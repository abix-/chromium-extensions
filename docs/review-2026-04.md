# Kovarex review. April 2026

Bar: a 10/10 codebase runs for 10 years, used daily by 100 people,
and never needs a code update. Anything less is not 10/10.

Today's grade: **6.5/10**. Real engineering is visible; the
10-year durability gap is not yet closed.

Companion doc: [todo.md](todo.md). Prioritized punch list
ordered by impact against this review's findings.

## The Good

Kept narrow because most of the review is criticism.

- `hush/src/lib.rs:9`. `#![forbid(unsafe_code)]`. Zero unsafe
  blocks in the entire Rust codebase. 10-year guarantee on
  memory safety.
- Only one `.unwrap()` in production code
  (`hush/src/background.rs:2123`), and it's guarded by an
  `is_none()` check at `:2117`. Safe today; ugly, see below.
- Zero TODOs / FIXMEs / HACKs across all three extensions.
  Exactly one `// ... for now ...` at
  `hush/src/suggestion.rs:27`. The discipline is uncommon.
- `hush/src/chrome_bridge.rs:26-28`.
  `serialize_maps_as_objects(true)`. Evidence that the
  `chrome.storage.local` silent-write-Map bug was encountered
  and fixed at the boundary, not papered over.
- `FoldHashState` at `hush/src/detectors.rs:19` with
  `AGG_MAP_CAPACITY = 16` at `:26`. Short-string keys, hot
  path, pre-sized aggregator. Performance thinking.
- `hush/Cargo.toml` release profile: `opt-level = 3`,
  `lto = "fat"`, `codegen-units = 1`, `panic = "abort"`,
  `strip = "symbols"`. Correct WASM ship profile.

## The Bad

### `background.rs:2123`. Guarded unwrap is a future bug

```rust
if tab_id.is_none() || key.is_empty() { ... return; } // :2117
let tab_id = tab_id.unwrap();                         // :2123
```

Safe today, one refactor to the guard kills it silently.
Idiomatic: `if let Some(tab_id) = tab_id`. Every `.unwrap()` in
non-test code is a failed-invariant check waiting for a refactor.

### `background.rs:430, 1021-1025, 1038-1040`. Silent failure

```rust
// :1021-1025  (on-installed bootstrap)
let _ = refresh_debug_flag().await;
let _ = seed_config_if_empty().await;
let _ = seed_allowlist_if_empty().await;
let _ = load_allowlist().await;
let _ = sync_dynamic_rules().await;
```

Any of five fails silently, user has a broken extension with
zero signal. `sync_dynamic_rules()` failing means the user's
Block rules don't fire. Same pattern at `:430` and
`:1038-1040`. 10-year codebase can't swallow errors on startup
paths. Minimum: `if let Err(e) = ... { log_error(...); }`.

### The four Hush monoliths

```
2499  src/background.rs
2362  src/ui_popup.rs
2057  src/ui_options.rs
1991  src/detectors.rs
```

No engineer holds 2500 LOC in their head. Every change costs
more than it should. Seams exist; cuts haven't been made. A
file nobody reads is a file that rots.

### `zoom-extension/content.js:200`. Forever-poll

```js
setInterval(() => {
  const video = getVideoElement();
  if (video && (video.style.transform !== ... || ...)) {
    applyZoom(video, currentOriginX, currentOriginY, false);
  }
}, 1000);
```

100 users × daily YouTube × 1 poll/sec × 10 years = hundreds of
billions of needless `document.querySelector("video")` calls.
Also runs on every YouTube page including ones with no video.
Should be a `MutationObserver` on the player container.

### `filter-anything-everywhere/extension/content.ts:234`. Unescaped regex from user input

```ts
const bannedWords = items.blacklist;
return regexpFromWordList(Object.keys(bannedWords));
```

User-entered words flow into `regexpFromWordList`. `word_matcher
.ts` must escape regex metachars. If it doesn't, entering a word
like `c++`, `a.b`, or `(test)` crashes the matcher or matches
nothing. Audit required.

### `zoom-extension/content.css`. YouTube-internal selectors

`.ytp-fullscreen-grid`, `.ytp-fullscreen-grid-main-content`,
`.ytp-fullscreen-grid-stills-container`,
`.ytp-fullscreen-grid-buttons-container`. YouTube renames
internal player markup approximately annually. Every rename
silently breaks the block until someone opens DevTools and
lands a PR. Opposite of "10 years untouched."

### Rust version pin is too aggressive

```toml
edition = "2024"
rust-version = "1.95"
```

Rust 1.95 is bleeding-edge. Debian stable lags ~1 year. Distro
builds from source break until toolchain catches up. 10-year
codebase: conservative MSRV like 1.80 based on actual dep needs.

### `hush/mainworld.js` at 913 LOC, zero tests

This is the **security-sensitive** file. Wraps `fetch`,
`XMLHttpRequest`, `sendBeacon`, `addEventListener`,
`Clipboard.prototype.readText`, `Bluetooth.requestDevice`,
prototype getters on `Navigator`, canvas + WebGL + audio,
EventTarget. One bug here breaks every site or leaks events that
shouldn't leak. Test count: **zero**. Verification by hand in a
browser. Unacceptable.

### `hush/src/stack.rs`. `Error().stack` string parsing

Script-origin attribution for every detector depends on parsing
V8 stack strings. The format is undocumented. V8 has changed it
before. When (not if) Chrome changes it again, every origin-
attributed detector silently stops attributing. Tests mock the
format; the runtime is at V8's mercy.

## Reinvented Wheels

### `rule_id` string format (`hush/src/types.rs::rule_id`)

```rust
format!("{action}::{scope}::{match_}")
```

A match containing `::` (e.g. `||foo::bar.example.com`) creates
an unreachable rule id. No version, no escape. A tuple key or a
serde-json'd struct would be equivalent effort and survive any
pattern.

### `canon::canonicalize_url`, `canon::pattern_keyword`

Hand-rolled URL logic in 155 lines. `url = "2.5"` is already a
dep. For eTLD+1 work, `publicsuffix` or `addr` exists. Worth
auditing how much of `canon.rs` is reinvention vs. legitimate
wrapper.

### `background.rs::schedule_persist_*` + `persist_*_scheduled` flags

Hand-rolled debounce around three persistence paths
(`:82-84` for the flags). `gloo-timers` is already pulled in
transitively by leptos. Not critical, but classic build-your-own.

### Hush's options + popup UIs in Leptos

~4500 LOC of Leptos components to render mostly tables.
Comparable extensions (including hush's own zoom-extension
options page at 34 LOC) ship plain DOM. The popup does not need
a reactive framework; the options page barely does. **Pre-1.0
framework dependency in load-bearing UI** is the single largest
durability risk in the repo.

## What Won't Survive 10 Years

| Threat | Where | Why it rots |
|---|---|---|
| **Leptos 0.x** | `hush/Cargo.toml`. `leptos = "0.8"` | Pre-1.0. Breaking changes every minor. 4500 LOC of UI tied to it. If Leptos pivots. They already refactored the reactive engine twice. The entire popup+options is a rewrite. |
| **Rust toolchain pin** | `hush/Cargo.toml:4-5` | `edition = "2024"`, `rust-version = "1.95"`. Bleeding-edge. Distros won't catch up for over a year. |
| **YouTube class names** | `zoom-extension/content.css` | YouTube renames internal markup annually. You already lived this. |
| **V8 stack format** | `hush/src/stack.rs`, `hush/mainworld.js::cap` | Undocumented; can change. Every origin-attribution detector breaks silently when it does. |
| **jQuery 4** | `filter-anything-everywhere/package.json` | jQuery's strategic future is uncertain. 3→4 already had breaking changes. More coming. |
| **filter-anything-everywhere upstream unmaintained** |. | No upstream security updates. Dep CVEs accumulate. 10-year out, `npm audit` fails. Either fully adopt the fork and maintain it, or it dies. |
| **MV3 itself** | All three `manifest.json` | Google's extension API surface. MV2 was deprecated. MV3 will be replaced. Not your fault; still name it. |
| **`rule_id` format** | `hush/src/types.rs::rule_id` | No version, no escape. `::` in a pattern = unreachable rule. |
| **`chrome.storage.local` schema with no `version`** | all reads/writes | Day 1 of a schema migration, you beg for a version number. Stamp it now. |

## What's Missing

1. **CI.** No `.github/workflows/`. 119 tests enforced by author discipline alone.
2. **Tests on `mainworld.js`**. 913 LOC of security-sensitive prototype-patching. Zero.
3. **Tests on `background.rs` handlers.** 2499 LOC of service-worker logic. Zero unit coverage on DNR sync, handler dispatch, persistence.
4. **Tests on `ui_popup.rs` / `ui_options.rs`**. ~4500 LOC of UI. Zero component tests.
5. **Recent bench baseline.** `docs/benchmarks.md` exists; four detectors shipped since the last recorded run (attention-tracking, clipboard-read, device-api-probe, navigator-fp). Perf regression invisible.
6. **Crash-recovery verification.** Service worker can be killed between ticks. `background.rs:82-84` debounced-persist pattern is racey under kill. No test proves durability.
7. **Regex metachar escape audit** on `filter-anything-everywhere/word_matcher.ts`.
8. **Schema version field** on everything in `chrome.storage.local`.
9. **Structured logging + request IDs.** Scattered `console.log` calls. Nothing correlates a detector firing to a rule hit to a DNR match.
10. **Permission audit.** `hush/manifest.json` grants several chrome.* permissions. Minimum-necessary justification for each, tracked over time, is absent.

## Where to Go Next. 10/10 ordering

Ordered by impact on "still runs in 10 years untouched, daily use
by 100 people."

1. **Dump Leptos from the load-bearing UI.** Rewrite popup +
   options as vanilla TS + DOM. Pre-1.0 framework dependency is
   the #1 durability risk. Keep simulate / firewall-log views if
   you genuinely need reactivity.
2. **Add GitHub Actions CI** for all three extensions.
3. **Write `mainworld.js` tests.** JSDOM or Node harness +
   prototype-spy. Cover install, call-through, spoof dedup
   (`hushSpoofEmitted`), and every branch.
4. **Write `background.rs` tests** for DNR sync +
   firewall-log persistence. Use `wasm-bindgen-test` + the
   existing `BackgroundState`.
5. **Audit and fix every `let _ = ` / `.ok()`** in
   `background.rs`. Log structured errors. Never silently drop
   on startup.
6. **Schema-version every `chrome.storage.local` value.** Wrap
   as `{ __hush_schema_version: 1, data: ... }`.
7. **Replace the `rule_id` string format** with a struct
   serialized via serde_json. No `::` ambiguity.
8. **Escape regex metachars** in `filter/word_matcher.ts`.
   Verify first, test, fix if broken.
9. **Kill the `zoom-extension` 1 Hz poll** (`content.js:200`).
   MutationObserver on body for `<video>` insertion.
10. **Dial down `rust-version`** to a conservative MSRV.
11. **Run and commit the Criterion baseline.** Gate future
    detector additions on no-regression.
12. **Self-test `Error().stack` format at runtime**. Log loud
    and bail gracefully if V8's format ever changes.
13. **Move YouTube selectors to a JSON config file** in
    zoom-extension so renames become data updates not code
    updates.
14. **Split the four Hush monoliths into submodules.**
    Readability compound interest.
15. **Fix `background.rs:2123` `.unwrap()`** for consistency.
16. **Write `filter-anything-everywhere/CHANGELOG.md`** tracking
    fork deltas vs upstream.

Items 1, 2, 3, 5, 6, 7 close the durability gap to ~8.5/10.
Items 4 and 11-13 get to 9. Rest is polish.

---

**Summary**: `#![forbid(unsafe_code)]`, disciplined guarding,
zero cruft markers, fold-hash aggregators, and the serializer
config fix all prove the engineering instinct is real. The
durability failures are architectural (Leptos, rule_id,
mainworld.js coverage) or startup-path (silent error swallows,
no schema version). Ship less. Test more. Delete more. Prefer
boring.
